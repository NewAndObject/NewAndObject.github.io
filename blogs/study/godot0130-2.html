状态机代码学习记录

悯 2026-01-30 22:01:00 Godot
  1. 创建4个脚本，分别是state.gd、player_state_machine.gd、state_idle.gd、state_walk.gd。分别的作用：state.gd 是模板类，用于被其他脚本继承，player_state_machine.gd用于管理协调state_idle.gd和state.walk.gd两个脚本的业务，而state_idle和state_walk.gd分别否则各种的业务
  2. 代码运行讲解
0. 创建4个脚本，分别是state.gd、player_state_machine.gd、state_idle.gd、state_walk.gd。分别的作用：state.gd 是模板类，用于被其他脚本继承，player_state_machine.gd用于管理协调state_idle.gd和state.walk.gd两个脚本的业务，而state_idle和state_walk.gd分别否则各种的业务。接下来按代码执行流程讲解
1. 当进入玩家场景，执行玩家脚本（player.gd）_ready()方法，该方法里执行玩家状态管理器脚本，参数为self，就是把玩家本身传过去（要在玩家脚本里添加静态的玩家状态管理器变量才可以调用）
2. 这时候，进入玩家状态管理器脚本的，获取到玩家参数，循环获取该玩家参数下的子节点（要在玩家节点下创建玩家状态管理器节点（类型为Node类型），并在该节点下添加两个子节点，一个是空闲idle和行走walk）的状态并存放在玩家状态管理器下的全局变量状态集合（该变量为数组Array类型），然后判断状态集合变量是否有数据（即判断size大小是否大于0），如果存在数据就调用改变状态方法ChangState()（这个方法下面讲解3），参数为状态集下第一个状态（第一个状态为状态管理节点下的第一个子节点：空闲子节点idle），最后将节点设置为继承（即：process_mode=Node.PROCESS_MODE_INHERIT，原因是在玩家状态管理器脚本的_ready方法里设置该节点禁止继承：process_mode=Node.PROCESS_MODE_DISABLED，原因是玩家状态管理器节点一开始是不使用的，只有在该节点下有一个或多个状态时启动，其实也不用写这个代码也行）
3. 改变状态方法ChangeState()，接受状态参数（这个状态参数是我们创建的状态模板，下面会介绍7.1），判断新状态参数是否为空或和当前状态相同（当前状态默认空），第一次执行，不为空，判断当前状态是否为null，第一次执行当前状态变量为null，就不执行：就调用当前状态的Exit方法退出该状态，接下来执行：将临时状态全局变量存下当前状态（这里暂时用不上不用管），当前状态变量获取新状态参数值，当前状态执行Enter()方法，进入空闲状态脚本的Enter方法（）
4. 空闲脚本里，玩家静态变量调用改变动画方法，传：idle
5. 玩家脚本，更新动画节点调用play（）方法，参数为动画名称，我们获取到是状态是idle，接下来是获取方向，调用玩家脚本下的获取动画方向方法AimDirection()方法
6. AimDirection方法，判断当前方向全局变量坐标是否为：(0,1) 即Vector2.DOWN还是（0，-1）即Vector2.UP还是side侧边，第一次肯定(0,0)即侧边，最后播放的动画名称为：idle_side
7. 接下来行走逻辑相同，就不多解释，看代码吧
7.1 状态模板脚本类，继承了Node节点，有一个变量6个方法，一个静态的玩家变量（用于继承该该模板的调用玩家方法，等在闲置状态代码和行走状态代码讲解），6个方法：_ready()、Exit()（该方法为玩家进入状态执行逻辑）、Exit()（当玩家退出该状态时发生）、Process()、Physical()、HandleInput()。

#玩家状态管理类
class_name PlayerStateMachine extends State
#状态集合变量
var states:Array[State]
#临时状态缓存变量，记录上一个状态情况
var prev_state:State
#当前状态变量
var current_state:State

# 内置函数，只执行一次，优先级弱于_process()和_physical_process()
func _ready():
	##当进入该节点时，该节点所有子节点设置无效
	process_mode=Node.PROCESS_MODE_DISABLED
	pass # Replace with function body.


# 内置程序周期函数，1/60秒执行，优先级弱于_physical_process()内置函数
func _process(delta: float):
	ChangeState(current_state.Process(delta))
	pass
	
# 内置程序周期函数，1/60秒执行
func _physics_process(delta: float) -> void:
	ChangeState(current_state.Physics(delta))
	pass
#内置函数，控制输入时调用
func _unhandled_input(event: InputEvent) -> void:
	ChangeState(current_state.HandleInput(event))
	pass
# 初始化方法，逻辑：
# 状态集合变量设置为空
# 循环获取该节点下的所有节点，并添加到状态节点变量里
# 判断状态集合变量是否为空，不为空就将参数玩家_player赋值给状态集合变量为索引0的玩家
# 启动节点，执行：process_mode=Node.PROCESS_MODE_INHERIT
func Intialize(_player:Player)->void:
	states=[]
	for c in get_children():
		if c is State:
			states.append(c)
	if states.size()>0:
		states[0].player=_player
		ChangeState(states[0])
		##将该节点下的子节点启动，设置为可继承，即默认设置
		process_mode=Node.PROCESS_MODE_INHERIT
		

# 改变状态方法
#逻辑：当参数为null或和当前状态相同，不操作，
#不同时，先退出之前的状态即当前状态变量current_state调用Exit()方法，然后将当前状态current_state存放在缓存变量pre_state里
#接着，将参数新变量new_state赋予当前状态变量current_state
#最后当前状态current_state调用进入状态方法Enter()
func ChangeState(new_state:State)->void:
	if new_state==null || new_state==current_state:
		return
	if current_state:
		current_state.Exit()
	prev_state=current_state
	current_state=new_state
	current_state.Enter()

class_name  State_Walk extends State

# 加速度变量
@export var move_seep : float =100.0

# 方法控制：当玩家进入该状态时发生
func Enter()->void:
	## 进入就调用静态的玩家变量的更新动画方法，并传行走值：walk
	player.UpdateAnimation("walk")
	pass
	
# 该方法控制：当玩家退出该状态时发生
func Exit()->void:
	pass

# 该方法控制：当_process发生期间时状态进行的操作
func Process(_delta:float)->State:
	## 当玩家位置没有移动即（0，0），就返回空置变量，
	## 否则，静态玩家变量位置设置（系统的位置变量：velocity）为玩家位置（我们的设置的位置变量：direction）乘于加速度变量move_speed
	## 判断玩家是否位置变化了，有就调用玩家更新动画方法，传行走：walk，并返回null
	if player.direction==Vector2.ZERO:
		return idle
	player.velocity=player.direction*move_seep
	
	if player.SetDirection():
		player.UpdateAnimation("walk")
	return null

# 动画类变量，快捷方法：可以按Ctrl+鼠标左键选中的节点托至代码里
@onready var animation_player : AnimationPlayer = $AnimationPlayer

@onready var sprite_2d : Sprite2D = $Sprite2D
@onready var state_machine: PlayerStateMachine = $StateMachine

#初始方向变量，默认向下
var cardinal_direction : Vector2 = Vector2.DOWN
#初始位置变量，默认（0，0）
var direction : Vector2 = Vector2.ZERO

# 内置函数，只执行一次，优先级弱于_process()和_physical_process()
func _ready():

# 内置程序周期函数，1/60秒执行，优先级弱于_physical_process()内置函数
func _process(delta):
	#print("velocity:",velocity)
	#获取xy轴坐标，获取方法是：正参数减负参数。在godot里，xy坐标轴中，x轴正坐标在右边的坐标轴上，y轴的正坐标在下面的坐标轴上
	direction.x=Input.get_action_strength("right")-Input.get_action_strength("left")
	direction.y=Input.get_action_strength("down")-Input.get_action_strength("up")
	
	pass
# 内置物理程序周期函数，1/60秒执行
func _physics_process(delta):
	#内置函数，更新移动坐标
	move_and_slide()
	pass
#设置方向方法，返回bool类型
func SetDirection() -> bool:
	#创建局部变量new_direction获取初始方向
	var new_direction:Vector2=cardinal_direction
	#判断获取了最新初始方向的局部变量是否有变化，没有返回false
	if direction==Vector2.ZERO:
		return false
	#判断，在y轴没有变化时，新的方向是否往左变化（即全局变量direction的x属性是否小于0），变化了方向为-1（也可以写成Vector2.LEFT），否则相反
	if direction.y==0:
		new_direction=Vector2.LEFT if direction.x<0 else Vector2.RIGHT
	#判断参考上
	if direction.x==0:
		new_direction=Vector2.UP if direction.y<0 else Vector2.DOWN
	#判断，新方向是否和初始方向相同，相同就返回false。这里可能会有疑惑：新方向不是在上面获取了初始方向的值嘛，肯定相同啊，我的思考结果是：新方向在代码58到62这里变化了
	if new_direction==cardinal_direction:
		return false
	#将新方向赋予初始方向，初始方向发生变化
	cardinal_direction=new_direction
	#左右翻转，因为左右只有side侧边动画一个，要实现左右动画可以把侧边动画沿y轴反转，利用精灵节点的scale属性
	sprite_2d.scale.x=-1 if cardinal_direction==Vector2.LEFT else 1
	return true

#更新动画方法
func UpdateAnimation(state:String) -> void:
	#调用动画节点的播放方法，方法参数为你创建的动画名字：idle_up、walk_down等。
	#因为动画每次按键动画有方向和状态变化，所以需要这两个参数时时变化，state状态参数变化参考SetState()方法，方向参数变化参考SetDirection()方法和Aimdirection()方法
	animation_player.play(state + "_"+AimDirection())
	pass
#动画方向获取方法，返回String
func AimDirection()->String:
	#获取最新的初始方向，如果方向向下，返回“down”，如果方向向上，返回“up”，否则返回侧面“side”
	if cardinal_direction==Vector2.DOWN:
		return "down"
	elif cardinal_direction==Vector2.UP:
		return "up"
	else:
		return "side"
