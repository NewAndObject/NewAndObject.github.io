C#学习总结笔记-基础

# 什么是C#

  1. 来源于C语言
  2. 面向对象语言
  3. 可在NET上运行(可称.NET Framerwork)
  4. Net(Net Framework):由公共语言运行时（CLR）的虚执行系统（也称：公共语言运行库）和一组类库
  5. CLR：是Micrsoft对公共语言基础结构（CLI）国际标准的实现
  6. CLI（命令运行界面）：可理解为用户通过键盘输入指令，计算机收到指令后，给予执行，有点像window的cmd界面，不能用鼠标操作，CLI是创建执行和开发环境的基础，语言和库可以在其中无缝地协同工作
  7. IL（中间语言）：是高级语言编译后的结果（生成要符合公共类型规范-CLS）
  8. CTS（公共类型规范）：是一个有.Net语言都支持的数据类型集合，确保这些语言可以相互传递数据，体现了.Net语言的互操作性
  9. 过程：高级语言源代码>>>中间语言>>>机器语言

# 什么是This

  • 可引用特定的封闭对象，可以理解为在某个类中，你this，就是调用这个类

# 什么是类型

  • 可以理解为一种模具，不会占内存

# 什么是变量

  • 可以理解为有了对象的模具，会占内存，变量有名字（标识符），可通过这个名字找到模具里有什么东西

# 类型分类

  1. 值类型
  • 值类型的特点：多对多，什么是多对多呢，就是你有一个有东西的模具，如果给了一个变量，就有一个模具，再给一个变量，就会又有一个模具，一个变量对应一个模具（这个变量是值类型）
  1. 引用类型
  • 引用类型的特点：一对多，还是一个模具，给了一个引用类型变量，是一个模具，再给一个变量，它还是一个模具。
  1. 关系：值类型的变量用箱子装了东西，如果想要变成引用类型，就得保箱子拆开吗，而引用类型要变成值类型，就得把拆开的箱子装起来，这两个过程就称之为拆箱和装箱
  2. 用专业的术语讲，就是每建一个值类型的变量，就会有一个内存来存放，有专门的指针指向内存，而引用类型，不管你建多少变量，它的指针还是一样的，会指向同一个内存
  3. 值类型分类
  • 简单类型
    • 有符整形：有+/- 符号，Sbyte
    • 无符类型：无有+/- 符号，byte，（short、int、long）加前缀，nuint、nuint
    • Unicode
    • 字符（char）
    • 布尔类型
    • IEEE二进制浮点
    • 高精度十进制浮点
  • 枚举类型（enum）：英文读法 厄儿了们儿
  • 结构类型（struct）:英文读法：死去儿
  • 可null值类型：就是再值类型后加？号
  • 元组类型：有点像ES6的解构赋值，如：（double，int） t=（4.5，3）
  1. 引用类型
    • 类类型
    • Unicode字符的String类型
    • 接口类型（interface）
    • 委托类型（delegate）
    • 数组类型

# 虚方法

  1. 英文：virtual 我去乐
  2. 作用：虚方法可以再子类被重写（overide）或重新赋值（new 该虚方法）

# 抽象方法

  1. 英文：abstract 阿布是缺可特
  2. 特点：
    • 与虚方法类似
    • 与虚方法区别
      • 虚方法有具体实现操作，抽象方法无
      • 虚方法子类可重写或覆盖亦无，而抽象方法子类必须有一个方法重写父类抽象方法
      • 虚拟方法无，而抽象方法必须在抽象类中

# 接口

  1. 英文：Interface 赢特飞死
  2. 什么是接口：比如，它像抽象方法/类一样，是一个模具，你向模具里到水泥，它形成的就水泥材质的东西，你倒入的是黄金，就是黄金材质的东西
  3. 接口和抽象的区别
    • 接口无构造方法，抽象无
    • 无普通变量，抽象有普通变量
    • 可有多个，抽象只能有一个
    • 全是抽象的方法， 抽象类可不全是抽象方法
    • 不能有静态方法，抽象类可有
    • 必须public，抽象类可public可prtectectol

# 重构（重写）、重载区别

  1. 重构有overide前缀，重载无
  2. 方法一般在子类中用，子类对父类的方法重写（改变方法内容），而重载是一般在类自己里面前行重载，通过不同参数重载

# 索引器

  1. 什么是所引器：与数组类似，但索引不局限是整形，其他类型都可以

# Out、Ref

  1. Out、Ref是什么：都是函数参数的前缀，都可以取经过函数运算的结构，但不同的是，Ref必须是已赋值的参数

# 构造函数

  1. 什么是构造函数：名字与类一样，但参数不同，名字区分大小写，在类里写

# 委托

  1. 英文名字：delegate
  2. 声明：public delegate className（int _name）{}
  3. 实例：自己百度
  4. 特点：参数可是方法（但方法返回类型要与委托参数类型一样）
  5. 具体像什么，怎么个运作
    • 比如，两个人，一个行为，一个快递员，一个收件人，一个发向何处的行为，快递员有多个（受委托的参数类型影响），发件如委托中通快递员工快递到北京去

# 运算符重载

  1. 关键词：operator
  2. 实例：
using System;
namespace OperatorOvlApplication
{
   class Box
   {
      private double length;      // 长度
      private double breadth;     // 宽度
      private double height;      // 高度

      public double getVolume()
      {
         return length * breadth * height;
      }
      public void setLength( double len )
      {
         length = len;
      }

      public void setBreadth( double bre )
      {
         breadth = bre;
      }

      public void setHeight( double hei )
      {
         height = hei;
      }
      // 重载 + 运算符来把两个 Box 对象相加
      public static Box operator+ (Box b, Box c)
      {
         Box box = new Box();
         box.length = b.length + c.length;
         box.breadth = b.breadth + c.breadth;
         box.height = b.height + c.height;
         return box;
      }

   }

   class Tester
   {
      static void Main(string[] args)
      {
         Box Box1 = new Box();         // 声明 Box1，类型为 Box
         Box Box2 = new Box();         // 声明 Box2，类型为 Box
         Box Box3 = new Box();         // 声明 Box3，类型为 Box
         double volume = 0.0;          // 体积

         // Box1 详述
         Box1.setLength(6.0);
         Box1.setBreadth(7.0);
         Box1.setHeight(5.0);

         // Box2 详述
         Box2.setLength(12.0);
         Box2.setBreadth(13.0);
         Box2.setHeight(10.0);

         // Box1 的体积
         volume = Box1.getVolume();
         Console.WriteLine("Box1 的体积： {0}", volume);

         // Box2 的体积
         volume = Box2.getVolume();
         Console.WriteLine("Box2 的体积： {0}", volume);

         // 把两个对象相加
         Box3 = Box1 + Box2;

         // Box3 的体积
         volume = Box3.getVolume();
         Console.WriteLine("Box3 的体积： {0}", volume);
         Console.ReadKey();
      }
   }
}

# 事件

  1. 关键词：event
  2. 特点：与委托一起使用
  3. 实例：
using System;
using System.IO;

namespace BoilerEventAppl
{

   // boiler 类
   class Boiler
   {
      private int temp;
      private int pressure;
      public Boiler(int t, int p)
      {
         temp = t;
         pressure = p;
      }

      public int getTemp()
      {
         return temp;
      }
      public int getPressure()
      {
         return pressure;
      }
   }
   // 事件发布器
   class DelegateBoilerEvent
   {
      public delegate void BoilerLogHandler(string status);

      // 基于上面的委托定义事件
      public event BoilerLogHandler BoilerEventLog;

      public void LogProcess()
      {
         string remarks = "O. K";
         Boiler b = new Boiler(100, 12);
         int t = b.getTemp();
         int p = b.getPressure();
         if(t > 150 || t < 80 || p < 12 || p > 15)
         {
            remarks = "Need Maintenance";
         }
         OnBoilerEventLog("Logging Info:\n");
         OnBoilerEventLog("Temparature " + t + "\nPressure: " + p);
         OnBoilerEventLog("\nMessage: " + remarks);
      }

      protected void OnBoilerEventLog(string message)
      {
         if (BoilerEventLog != null)
         {
            BoilerEventLog(message);
         }
      }
   }
   // 该类保留写入日志文件的条款
   class BoilerInfoLogger
   {
      FileStream fs;
      StreamWriter sw;
      public BoilerInfoLogger(string filename)
      {
         fs = new FileStream(filename, FileMode.Append, FileAccess.Write);
         sw = new StreamWriter(fs);
      }
      public void Logger(string info)
      {
         sw.WriteLine(info);
      }
      public void Close()
      {
         sw.Close();
         fs.Close();
      }
   }
   // 事件订阅器
   public class RecordBoilerInfo
   {
      static void Logger(string info)
      {
         Console.WriteLine(info);
      }//end of Logger

      static void Main(string[] args)
      {
         BoilerInfoLogger filelog = new BoilerInfoLogger("e:\\boiler.txt");
         DelegateBoilerEvent boilerEvent = new DelegateBoilerEvent();
         boilerEvent.BoilerEventLog += new
         DelegateBoilerEvent.BoilerLogHandler(Logger);
         boilerEvent.BoilerEventLog += new
         DelegateBoilerEvent.BoilerLogHandler(filelog.Logger);
         boilerEvent.LogProcess();
         Console.ReadLine();
         filelog.Close();
      }//end of main

   }//end of RecordBoilerInfo
}

# 多线程

  1. 关键词：System.Threading.Thread
  2. 实例：
using System;
using System.Threading;

namespace MultithreadingApplication
{
    class ThreadCreationProgram
    {
        public static void CallToChildThread()
        {
            try
            {

                Console.WriteLine("Child thread starts");
                // 计数到 10
                for (int counter = 0; counter <= 10; counter++)
                {
                    Thread.Sleep(500);
                    Console.WriteLine(counter);
                }
                Console.WriteLine("Child Thread Completed");

            }
            catch (ThreadAbortException e)
            {
                Console.WriteLine("Thread Abort Exception");
            }
            finally
            {
                Console.WriteLine("Couldn't catch the Thread Exception");
            }

        }
       
        static void Main(string[] args)
        {
            ThreadStart childref = new ThreadStart(CallToChildThread);
            Console.WriteLine("In Main: Creating the Child thread");
            Thread childThread = new Thread(childref);
            childThread.Start();
            // 停止主线程一段时间
            Thread.Sleep(2000);
            // 现在中止子线程
            Console.WriteLine("In Main: Aborting the Child thread");
            childThread.Abort();
            Console.ReadKey();
        }
    }
}
