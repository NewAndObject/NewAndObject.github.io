godot2D游戏开发流程

悯 2026-04-25 17:30:00 godot

# 让人物动起来：2D坐标向量-Vector2D

  1. 创建主场景，节点类型：Node2D
  2. 创建移动物体-玩家场景，节点类型：CharacterBody2D
  3. 给玩家下创建子节点，节点类型：CollisionShape2D-碰撞节点，让玩家可以和其他物体发生碰撞。和移动没关系
  4. 玩家下创建精灵节点-Sprite2D，并将多帧的图片拖到Texture属性上
  5. 在精灵节点上修改Hframes-动画列数和Vframes-动画行数
  6. 创建玩家节点脚本
  7. 项目创建输入映射：项目-项目设置-输入映射-添加新动作，编辑新动作输入动作映射的按键等
  8. 在_process函数内获取映射的动作输入变化，方法有多种，例如：
velocity=Vector2(
		Input.get_axis("left","right"),
		Input.get_axis("up","down")
	)
##velocity是CharacterBody2D的速度矢量属性，一般表现为二维坐标即属性类型为Vector2D

# 让世界多彩：TileMapLayer

  1. 创建TileMapLayer节点
  2. 将地图瓦片集图片拖到图块里
  3. 在Ordering里设置Z Index，不要和玩家处于同一层
  4. 设置Tile Set的Tile Size，为你的地图图片的最小单位像素大小
  5. 点击Tile Set 属性，设置Physics Layers的Collision Layer 为对应为主。Terrain Sets 属性的Mode属性下拉框选择Match Corners And Sides，创建一个Terrains，Name任意，Color默认
  6. 找到TileSet栏，绘制>下拉框选择-地形 绘制Terrain Set 下拉框选择Terain Set0，Terrain属性下拉框选择Path0（5步骤创建的Name），然后在右边的地图里点出瓦片集
  7. 找到TileSet栏，绘制>下拉框选择-物理层 在右边的图片里选择碰撞区域
  8. 最后将创建的TileMapLayer节点设置为场景，然后在需要的地方Node2D节点下放置就可以使用该场景的瓦片集和图片了

# 动画节点动起来

  1. 在玩家场景下场景AnimactionPlayer节点
  2. 点击动画栏-动画创建动画
  3. 点击动画栏-+创建轨道->选择属性轨道->选择你要创建动画的精灵节点（sprite节点）->选择该节点下的frame属性（帧属性）
  4. 根据需求创建多个帧属性轨道，例如：上下左右的待机帧轨道和上下左右的行走帧轨道
  5. 在多个帧轨道里选择一个作为游戏加载后的自动加载动画
  6. 帧轨道有在动画栏右边有个循环类型：不循环、循环、来回循环
  7. 修改玩家脚本。
    1. 先根据在_process里获取的移动坐标向量判断它当前是上下左右那个方向，
    2. 再根据移动坐标向量判断是否移动判断是行走还是空闲那个状态，
    3. 最后更新动画（调用动画节点的play方法更新动画）

# 总结

动画帧创建没什么说的，说一下脚本逻辑，主要靠获取的移动的坐标向量判断方向和状态，最后更新动画

  1. 创建状态管理器脚本，继承状态基类
#玩家状态管理类
class_name PlayerStateMachine extends State
#状态集合变量
var states:Array[State]
#临时状态缓存变量，记录上一个状态情况
var prev_state:State
#当前状态变量
var current_state:State

# 内置函数，只执行一次，优先级弱于_process()和_physical_process()
func _ready():
    ##当进入该节点时，该节点所有子节点设置无效
    process_mode=Node.PROCESS_MODE_DISABLED
    pass # Replace with function body.


# 内置程序周期函数，1/60秒执行，优先级弱于_physical_process()内置函数
func _process(delta: float):
    ChangeState(current_state.Process(delta))
    pass
    
# 内置程序周期函数，1/60秒执行
func _physics_process(delta: float) -> void:
    ChangeState(current_state.Physics(delta))
    pass
#内置函数，控制输入时调用
func _unhandled_input(event: InputEvent) -> void:
    ChangeState(current_state.HandleInput(event))
    pass
# 初始化方法，逻辑：
# 状态集合变量设置为空
# 循环获取该节点下的所有节点，并添加到状态节点变量里
# 判断状态集合变量是否为空，不为空就将参数玩家_player赋值给状态集合变量为索引0的玩家
# 启动节点，执行：process_mode=Node.PROCESS_MODE_INHERIT
func Intialize(_player:Player)->void:
    states=[]
    for c in get_children():
        if c is State:
            states.append(c)
    if states.size()>0:
        states[0].player=_player
        ChangeState(states[0])
        ##将该节点下的子节点启动，设置为可继承，即默认设置
        process_mode=Node.PROCESS_MODE_INHERIT
        

# 改变状态方法
#逻辑：当参数为null或和当前状态相同，不操作，
#不同时，先退出之前的状态即当前状态变量current_state调用Exit()方法，然后将当前状态current_state存放在缓存变量pre_state里
#接着，将参数新变量new_state赋予当前状态变量current_state
#最后当前状态current_state调用进入状态方法Enter()
func ChangeState(new_state:State)->void:
    if new_state==null || new_state==current_state:
        return
    if current_state:
        current_state.Exit()
    prev_state=current_state
    current_state=new_state
    current_state.Enter()

class_name Player extends CharacterBody2D

# 动画类变量，快捷方法：可以按Ctrl+鼠标左键选中的节点托至代码里
@onready var animation_player : AnimationPlayer = $AnimationPlayer
# 精灵类变量，快捷方法如上
@onready var sprite_2d : Sprite2D = $Sprite2D
@onready var state_machine: PlayerStateMachine = $StateMachine

#初始方向变量，默认向下
var cardinal_direction : Vector2 = Vector2.DOWN
#初始位置变量，默认（0，0）
var direction : Vector2 = Vector2.ZERO

# 内置函数，只执行一次，优先级弱于_process()和_physical_process()
func _ready():
    state_machine.Intialize(self)
    pass 

# 内置程序周期函数，1/60秒执行，优先级弱于_physical_process()内置函数
func _process(delta):
    #print("velocity:",velocity)
    #获取xy轴坐标，获取方法是：正参数减负参数。在godot里，xy坐标轴中，x轴正坐标在右边的坐标轴上，y轴的正坐标在下面的坐标轴上
    direction.x=Input.get_action_strength("right")-Input.get_action_strength("left")
    direction.y=Input.get_action_strength("down")-Input.get_action_strength("up")
    
    pass
# 内置物理程序周期函数，1/60秒执行
func _physics_process(delta):
    #内置函数，更新移动坐标
    move_and_slide()
    pass
#设置方向方法，返回bool类型
func SetDirection() -> bool:
    #创建局部变量new_direction获取初始方向
    var new_direction:Vector2=cardinal_direction
    #判断获取了最新初始方向的局部变量是否有变化，没有返回false
    if direction==Vector2.ZERO:
        return false
    #判断，在y轴没有变化时，新的方向是否往左变化（即全局变量direction的x属性是否小于0），变化了方向为-1（也可以写成Vector2.LEFT），否则相反
    if direction.y==0:
        new_direction=Vector2.LEFT if direction.x<0 else Vector2.RIGHT
    #判断参考上
    if direction.x==0:
        new_direction=Vector2.UP if direction.y<0 else Vector2.DOWN
    #判断，新方向是否和初始方向相同，相同就返回false。这里可能会有疑惑：新方向不是在上面获取了初始方向的值嘛，肯定相同啊，我的思考结果是：新方向在代码58到62这里变化了
    if new_direction==cardinal_direction:
        return false
    #将新方向赋予初始方向，初始方向发生变化
    cardinal_direction=new_direction
    #左右翻转，因为左右只有side侧边动画一个，要实现左右动画可以把侧边动画沿y轴反转，利用精灵节点的scale属性
    sprite_2d.scale.x=-1 if cardinal_direction==Vector2.LEFT else 1
    return true

#更新动画方法
func UpdateAnimation(state:String) -> void:
    #调用动画节点的播放方法，方法参数为你创建的动画名字：idle_up、walk_down等。
    #因为动画每次按键动画有方向和状态变化，所以需要这两个参数时时变化，state状态参数变化参考SetState()方法，方向参数变化参考SetDirection()方法和Aimdirection()方法
    animation_player.play(state + "_"+AimDirection())
    pass
#动画方向获取方法，返回String
func AimDirection()->String:
    #获取最新的初始方向，如果方向向下，返回“down”，如果方向向上，返回“up”，否则返回侧面“side”
    if cardinal_direction==Vector2.DOWN:
        return "down"
    elif cardinal_direction==Vector2.UP:
        return "up"
    else:
        return "side"

# 总结

  1. 玩家脚本ready里调用状态管理器的初始化，将玩家self传过去

  2. 而初始化中是获取状态管理器的子节点（行走、空闲子节点）

  3. 再将第一个状态节点传给状态管理器里的改变状态方法，参数是我们创建的状态基类（所以一般空闲状态放在状态管理器下第一个位置），

  4. 然后调用空闲节点的进入状态方法，方法里再调用玩家的更新动画方法，将动状态名称传过去。在空闲节点里判断玩家是否移动，是就返回行走类型，

  5. 等状态管理器的_process调用当前状态的Process方法，

  6. 如果是空闲状态节点的脚本里的Process方法，会判断玩家里的位置向量是否变化，变化就返回行走状态类型，反之什么也不返回

  7. 如果返回行走状态类型，在状态管理器代码的_process方法里进行下一步会调用状态管理器下脚本下的改变状态方法。如果状态和之前的状态一样，什么也不干，反之就是行走状态就等于这个状态的进入方法，进入方法会调用玩家的动画更新方法，参数就是行走名称

# 互动的世界：Area2D和CollisionObject2D

  1. 互动的本质是，比如，两个精灵节点（A和B）下分别各自创建一个CollisionShpae2D节点和Area2D节点，当A精灵的CollisionSpape2D进入另一个B精灵的Area2D范围，触发B精灵的Area2D的信号，然后B精灵做出反应，A同理。
  2. 打怪的世界：Area2D、CollisionObject2D和AnimactionPlayer
    1. 在互动的基础上，添加动画的变化，即在B精灵检查到A精灵进入自己的区域，触发动画节点AnimationPlayer
    2. 战斗交互：Area2D
      1. 在玩家场景里添加攻击的动画轨道
      2. 状态管理器节点下添加攻击节点和脚本
      3. 添加通用场景-盒子场景，用于绑定在被攻击的物体上，比如敌人场景下，节点类型：Area2D，开启monitorable属性（开启了该属性，可以被其他Area2D节点检测到）
      4. 添加通用场景-攻击场景，用于绑定在攻击的节点，比如玩家场景下，节点类型：Area2D，开启monitoring属性（开启了该属性会检测经过Area2D下的区域）
      5. 创建被攻击创建，比如植物，先创建一个Node2D节点，节点下创建基类节点，并绑定盒子场景为子节点，另外还需要创建一个StaticBody2D节点在植物场景下，起到固定不可移动作用
      6. 玩家场景下玩家精灵节点下绑定攻击盒子场景为子节点

    • 总结

      1. 状态机-攻击：开启攻击区域节点是否可以检测进入攻击区域的属性->攻击盒子：连接Area2D的检测区域信号，如果检测到被攻击的盒子，就告诉盒子你被攻击了->被攻击的盒子：得知被攻击了，就发出我被攻击了的信号告知植物（即它的捆绑的植物精灵节点）->植物得知后，掉血死亡消失
      2. 当玩家的攻击植物时，在state_attack里将伤害盒子的monitoring开启，告诉他可以检测到其他盒子了，
      3. 伤害盒子脚本的_ready里连接area_entered.connect伤害盒子进入其他区域信号，调用本地方法（AreaEntered()）方法检测
      4. AreaEntered判断进入的区域Area2D是否是HitBox类型节点，是就调用盒子节点受到伤害方法TakeDamage()并将self传过去（伤害盒子传过去，也可以传伤害值过去），
      5. 在盒子脚本里，TakeDamage里接受到伤害类参数，调用盒子的伤害信号Damage，并将伤害类参数传过去
      6. 最后，在植物节点脚本里，获取盒子节点的Damage信号并连接调用本地方法，告知植物被伤害到了，需要消失或掉血

# 让世界有声音：AudioStreamPlayer2D节点
