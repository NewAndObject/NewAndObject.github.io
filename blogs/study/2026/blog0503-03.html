# 核心配置文件

# 1. package.json

项目配置文件，定义了项目的名称、版本、依赖和脚本命令。

{
  "name": "msj-blog",
  "version": "1.0.0",
  "scripts": {
    "dev": "vuepress dev .",
    "build": "vuepress build ."
  },
  "devDependencies": {
    "vuepress": "1.5.0",
    "vuepress-theme-reco": "1.5.5"
  }
}

# 1. config.js

站点核心配置文件，控制整个网站的行为和外观。

# 主要配置项

module.exports = {
  "title": "悯",                    // 网站标题
  "description": "光阴似箭",        // 网站描述
  "dest": "public",                 // 构建输出目录
  "theme": "reco",                  // 使用的主题
  "head": [...],                    // 注入到 <head> 的标签
  "themeConfig": {                  // 主题配置
    "nav": [...],                   // 导航栏菜单
    "sidebar": {...},               // 侧边栏配置
    "type": "blog",                 // 博客模式
    "blogConfig": {...},            // 博客功能配置
    "friendLink": [...],            // 友情链接
    "author": "悯",                 // 作者名称
    "logo": "/school.jpg",          // 导航栏 logo
    "search": true,                 // 搜索功能
  },
  "markdown": {
    "lineNumbers": true             // 代码块显示行号
  },
  "plugins": [...]                  // 插件配置
}

# 2. enhanceApp.js

客户端增强文件，用于在浏览器端执行自定义逻辑。

# 主要用途

用途 说明
注册全局组件 Vue.component('MyComponent', ...)
路由钩子 router.afterEach(...)
注入第三方脚本 如 Giscus 评论系统

# 示例：Giscus 注入

export default ({ router, isServer }) => {
  if (!isServer) {
    router.afterEach(() => {
      setTimeout(() => {
        // 动态加载 Giscus 脚本
      }, 1000)
    })
  }
}

# 其他文件

文件 说明
.gitignore Git 忽略规则，不提交 node_modules/、public/ 等
package-lock.json npm 依赖版本锁定文件
yarn.lock yarn 依赖版本锁定文件
node_modules/ 依赖包目录，不提交到 Git

# 常用命令

命令 说明
npm run dev 启动本地开发服务器
npm run build 构建静态网站到 public/ 目录

# 总结

目录/文件 核心作用
.vuepress/config.js 配置网站标题、主题、导航、插件
.vuepress/enhanceApp.js 客户端增强，如注入脚本、注册组件
.vuepress/public/ 静态资源，如图片、favicon
blogs/ 博客文章，按分类和年份组织
README.md 首页内容配置
public/ 构建输出目录，用于部署
