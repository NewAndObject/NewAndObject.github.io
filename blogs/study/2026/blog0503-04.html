# VuePress 博客添加 BGM：vuepress-plugin-bgm-player 配置指南

在使用 vuepress-theme-reco 主题搭建博客时，添加一个优雅的背景音乐播放器能显著提升访客体验。@vuepress-reco/vuepress-plugin-bgm-player 是该主题的官方插件，基于 HTML5 Audio 开发，支持播放列表、自动收缩、悬浮窗模式等多种高级功能。

本文将详细讲解该插件的配置步骤及在实际使用中容易遇到的坑。

# 一、 安装插件

在博客项目根目录下，使用 npm 或 yarn 安装插件：

npm install @vuepress-reco/vuepress-plugin-bgm-player -D
1

# 二、 配置过程

打开 .vuepress/config.js，在 plugins 数组中添加插件配置。

# 1. 基础配置示例

module.exports = {
  // ... 其他配置
  plugins: [
    // ... 其他插件
    [
      "@vuepress-reco/vuepress-plugin-bgm-player",
      {
        // 核心：音频列表
        audios: [
          {
            name: "海浪白噪音",
            artist: "自然之声",
            // 网络音频链接
            url: "https://cdn.pixabay.com/audio/2022/03/15/audio_c8c8a73467.mp3",
            cover: "https://example.com/cover.jpg"
          }
        ],
        // 播放器位置
        position: {
          left: "10px",
          bottom: "10px",
          "z-index": "999999"
        },
        // 是否自动播放（多数浏览器默认禁止）
        autoplay: false,
        // 是否自动缩小
        autoShrink: false
      }
    ]
  ]
};

# 2. 核心参数详解

参数 说明 类型 默认值 备注
audios 播放列表 Array - 必填，支持本地/网络音频
position 播放器位置 Object 左下角 可自定义 top/right/bottom/left
autoplay 自动播放 Boolean false 现代浏览器通常拦截静音外的自动播放
autoShrink 自动缩小 Boolean false 页面加载后是否自动折叠为迷你模式
shrinkMode 收缩模式 String float float（悬浮窗）或 mini（极简小图标）
floatPosition 悬浮位置 String left 当模式为 float 时生效，可选 left/right
floatStyle 悬浮窗样式 Object - 自定义悬浮窗 CSS，如 bottom

# 三、 音频来源配置

# 方式 1：使用网络音频

直接使用稳定的 CDN 链接或对象存储链接（如阿里云 OSS、七牛云等）。

audios: [
  {
    name: "歌曲名",
    artist: "歌手",
    url: "https://assets.smallsunnyfox.com/music/2.mp3",
    cover: "https://assets.smallsunnyfox.com/music/2.jpg"
  }
]

# 方式 2：使用本地音频

如果你有自己的 MP3 文件，可以按照以下步骤操作：

  1. 放置文件：将 MP3 文件放入 .vuepress/public/music/ 目录下（如果目录不存在，手动创建）。
  2. 配置路径：路径以 / 开头，对应 public 目录。

配置示例：

audios: [
  {
    name: "背景音乐",
    artist: "本地音乐",
    // 对应 .vuepress/public/music/bgm.mp3
    url: "/music/bgm.mp3",
    cover: "/music/cover.jpg"
  }
]

# 四、 注意事项与排错指南

在配置和使用过程中，以下问题最为常见，请务必注意：

# 1. 文件名避免中文和特殊字符

问题：使用中文文件名（如 【二胡】曲.mp3）或包含空格、括号的文件名，可能导致服务器解析路径失败（404）。 解决：

  • 将文件名改为纯英文或拼音（例如：erhu-background.mp3）。
  • 确保 config.js 中的 url 与实际文件名完全一致。

# 2. 音频编码格式问题

报错：NotSupportedError: The element has no supported sources. 原因：

  • 虽然文件后缀是 .mp3，但内部编码可能不被浏览器（特别是 Chrome/Edge）支持。
  • 文件可能损坏。 解决：
  • 使用工具（如格式工厂、Audacity）将文件重新转码为标准的 MPEG-1 Layer 3 (通常 128kbps, 44100Hz)。
  • 确保文件在本地能用播放器正常播放。

# 3. 浏览器自动播放限制

现象：设置 autoplay: true 后依然没声音。 原因：Chrome 等现代浏览器的自动播放策略要求用户必须先在页面上有过交互（点击、触摸等），否则禁止带声音的自动播放。 解决：

  • 接受现状：保留 autoplay: false，让用户手动点击播放。
  • 或者尝试静音自动播放（但这违背了 BGM 的初衷）。

# 4. 必须重启 Dev Server

修改 config.js 或添加新的静态资源（如放入 mp3 文件）后，必须完全停止并重启 npm run dev 才能生效。

# 5. 检查 404 问题

如果播放失败，打开浏览器控制台 (F12) -> Network 面板：

  • 刷新页面，点击播放。
  • 查看 mp3 请求的状态码。如果是 404，说明路径错误，请检查文件是否真的在 .vuepress/public/ 下，且路径以 / 开头。

# 6. 无法直接访问本地磁盘资源

注意：如果你在配置中直接填入了电脑本地的绝对路径（例如 file:///C:/xxx.mp3 或 D:/music.mp3），是无法播放音乐的。 原因：浏览器出于安全策略，禁止网页代码（包括本地运行的 Dev Server）直接通过绝对路径或 file:// 协议读取你电脑硬盘上的图片和音频资源。 解决：请务必将音乐/图片文件放入项目的 .vuepress/public/ 目录中，并使用以 / 开头的相对路径进行引用（例如 /music.mp3）或直接发布到GitHub看看是否可以播放
