  • 创建信号的三种方法
编辑器创建
	第一步：选中需要创建信号的节点
	第二步：选择右边的节点编辑模板
	第三步：选择你的信号并双击
	第四步：选择另外一个你要通信的节点
	第五步：在另一个节点会自动创建通信代码
	第六步：在创建的通信代码里写业务逻辑
手写代码创建
	第一步：创建func _ready()方法
	第二步：用一个变量存放使用get_node(“节点名称”)方法获取的节点
		get_node
	第四步：以time为例，使用timeout属性调用connect方法将业务逻辑方法作为参数
	第五步：创建业务逻辑代码方法
自定义代码创建
	第一步：extends 节点类
	第二步：创建通信：signal 自定义通信名称
	第三步：创建业务逻辑方法，最后通过 自定义通信名称.emit()进行发出信息
	第四步：和编辑器创建通信一样了

  • C#语言开发问题
1.如果用godot开发，按官方文档操作hellow world部分代码报错，其实他们也没写对
如果创建的是Sprite2D节点，就必须在类里重写_Ready里输出：
例如：
using Godot;
using System;

public partial class Sprite2d : Sprite2D
{
	// Called when the node enters the scene tree for the first time.
	public override void _Ready()
	{
		GD.Print("Hellow World!");
	}

	// Called every frame. 'delta' is the elapsed time since the previous frame.
	public override void _Process(double delta)
	{
	}
}

而在创建脚本时，模板选择Node：Default最好，它会默认添加_Ready()这部分代码，不然你就得自己写这部分代码了

另外，我好奇的是，_Ready()和_Process(double delta)这部分我知道是重写父级的方法，但是还是不理解，他们的的作用是初始化时执行和每帧执行吗？得找一下教程。
回答一下刚刚的两个问题：
第一：_Ready()是否是只加载一边，答案：是的
第二：_Process是否是每帧都加载，答案：是的
具体参考接口：Node的方法描述
https://docs.godotengine.org/zh-cn/4.x/classes/class_node.html#id4
