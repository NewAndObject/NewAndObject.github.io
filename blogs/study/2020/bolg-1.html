# 静态文件生成

  1. 打开cmd控制台
  2. 输入npx @vuepress-reco/theme-cli init my-blog(自定义文件名)
  3. 控制台访问生成的文件夹路径位置，在输入npm install
  4. 打开vs.code访问生成的文件
  5. 右键文件，选择open in Intergrated Tarminal
  6. 在下面的TERMINAL里输入npm run build 生成pubil静态界面

# GitHub静态网站部署方法

  1. 在GitHub网站注册一个账户
  2. 在账户里创建一个public的仓储
仓库创建注意事项:
1. 仓库名称格式：名字.github.io
2. 现在公开的:public
3. 勾选创建CNAME文件,创建一个CNAME用于网站访问（如果有域名，就需要将域名放在Custom domain里,并且CNAME里的值改成域名）
4. 勾选add .gitgnore,下拉框搜索Node并选中

  1. 上传方法2
1.安装git
2.配置ssh
    先生成ssh key
ssh-keygen -t rsa -C "1429398308@qq.com"
备注：这里的邮件地址是github账号绑定的邮件地址
输入生成命令：ssh-keygen -t rsa -C "邮件地址"后。连续回车三次即可(第一次是输入SSH文件名称，第二次是密码，第三次是确认密码，注意名称要是默认的，否则你执行ssh -T git@github.com时，会报：git@github.com: Permission denied (publickey).)

3. github 设置ssh key
    打开github主页，点击个人设置，点击左侧的SSH and GPG keys，点击New SSH key ,
密钥选择authentication key ,选择这个类型,是因为安全,可以每次你操作库需要密码,这样就不用担心被别人修改代码库,
然后将生成在C:\Users\Admin\.ssh文件夹里的id_rsa.pub复制的内容粘贴到key中，title随便起一个就行。

4. 测试是否成功
    ssh -T git@github.com
输入测试命令后，接着输入yes，出现下面这个提示说明成功

5. 配置账号和密码
    git config --global user.name "NewAndObject" #你的github用户名
git config --global user.email "1429398308@qq.com" #填写你的github注册邮箱

6. git 首次上传流程
    · 打开git命令面板，cd 文件夹路径
    · 输入git init
    · 输入git add .
    · 输入git commit -m "备注"   （第一次需要输入git邮箱和用户名）
    · 添加远程仓库地址：git remote add origin git@github.com:NewAndObject/NewAndObject.github.io.git（如果是已经创建了就不需要，直接执行下一步上传指令）
    · 上传指令：git push -u origin main  （main 是你GitHub主分支名称,origin 是一个默认的远程仓库名称。当你克隆一个远程仓库时，Git 会自动创建一个指向该远程仓库的别名，即 origin。）
    备注:上传代码之前记得拉取，不然报错：error: failed to push some refs to 'github.com:NewAndObject/NewAndObject.github.io.git'
    
7. git 新用户拉取更新流程
    · cd 文件夹路径
    . 拉取代码： git clone git@github.com:NewAndObject/NewAndObject.github.io.git(到GitHub里登录获取ssh指令)
    · 上传指令：git push -u origin main 
8. gti 老用户更新流程
    · cd在要上传的文件夹下
    . git add .
    . git commit -m "备注"
    . git push -u origin main (输入ssh密码,主分支，github是main，gitee是master)
